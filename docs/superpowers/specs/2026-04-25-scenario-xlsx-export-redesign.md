# Scenario xlsx export — редизайн «Бухгалтерия»

**Дата:** 2026-04-25
**Заменяет:** `2026-04-25-scenario-xlsx-export-design.md` (тот вариант с обложкой и cyan-карточками отвергнут — слишком декоративно)
**Endpoint:** `GET /api/v1/exports/scenarios/{id}.xlsx` (неизменно)
**Сервис:** `ScenarioXlsxExporter.build()` (rewrite)

## Контекст

Прежний вариант (7 листов с титульной обложкой, карточки-метрики, прогресс-бар, тёмная палитра под UI) PM забраковал — слишком «дашбордно» и декоративно для файла, в котором с ним работают, а не любуются. После прогона через визуальный браузер выбраны:

- **Стиль «Бухгалтерия»** — минимум украшений, всё в таблицах, цвет только в условном форматировании (отклонения, итоги, дефицит). Тёмная горизонтальная шапка-полоска `#1f2937` вместо тёмного фона. Шрифт Calibri 10–11pt, табличный.
- **Тонкая книга — 4 листа** вместо 7. Сводный «дашборд на одном листе» первым, затем включено / не вошло, в конце — справочник (правила + внешний QA + отсутствия).
- **Колоночный набор «Mid-11»** для инициатив — без технических полей вроде `cost_type`, `opo_analyst_ratio`, `involvement_coefficient`.

Этот спек заменяет всю реализацию `app/services/scenario_xlsx_export.py`. Endpoint и API не меняются.

### Типографика

- Шрифт всей книги: `Calibri`, размер по умолчанию 11pt.
- Шапки таблиц: 11pt bold, белый.
- Числа: `tabular-nums` через явный `number_format` (`#,##0`, `#,##0.0`, `0%`).
- Подзаголовки разделов (`section-header`): 11pt bold uppercase, фон `#F3F4F6`, граница снизу 2pt тёмно-серая.

### Общие правила оформления

- **Сетка скрыта** на всех листах (`sheet_view.showGridLines = False`).
- Границы ячеек рисуются только в шапках таблиц и итоговых строках.
- Заморозка: первая строка с шапкой таблицы (на каждом листе).
- Автофильтр на табличных листах — на всю таблицу.
- Колонки фиксированной ширины (под содержимое; `bestFit` не используется).
- Имя файла в `Content-Disposition`:
  `scenario_{quarter}_{year}_{slug}_{YYYY-MM-DD}.xlsx` — как в прежнем спеке.

## Структура книги: 4 листа

| # | Лист | Назначение |
|---|------|------------|
| 1 | Сводка | Метаданные, ключевые метрики, по ролям × месяцам, по сотрудникам |
| 2 | Включено | Полный список включённых инициатив, 11 колонок |

## Лист 1: Сводка («B3 — дашборд на одном листе»)

Первый и главный лист. Открыл файл — увидел всё важное за один скролл.

### Структура (сверху вниз)

**Ячейка A1: заголовок-полоска** (тёмно-серая шапка)
```
A1: Сценарий: {scenario.name} — {статус}        (мерджится A1:H1)
```
- Шрифт: Calibri 12pt bold, белый, фон `#1F2937`, высота строки 22pt.
- `{статус}`:
  - `черновик` — без декорации (просто слово).
  - `утверждён` — с галочкой `✓ утверждён` зелёным жирным (внутри той же ячейки или соседней).

**Ячейка A2: подзаголовок**
```
A2: Q2 2026 · команда «Аналитики Альфа» · сформировано 25.04.2026
```
- Calibri 10pt regular, серый `#6B7280`, фон `#F3F4F6`, мерджится A2:H2.

**Раздел «Сводка»** (начинается с A4)
- Section-header `СВОДКА` в A4, мерджится A4:H4.
- Таблица: 2 строки × 8 колонок, label/value/label/value/label/value/label/value:

| | A | B | C | D | E | F | G | H |
|--|--|--|--|--|--|--|--|--|
| 5 | Ёмкость, ч | 2 480 | План, ч | 2 240 | Остаток, ч | 240 | Использование | 90% |
| 6 | Включено, шт. | 18 | Не вошло, шт. | 7 | QA дефицит, ч | −40 | Отсутствия, ч | 120 |

- Label-ячейки (A, C, E, G): regular 10pt, выравнивание влево.
- Value-ячейки (B, D, F, H): bold 11pt, вправо, формат `#,##0`.
- «QA дефицит» и любое отрицательное значение остатка/дефицита — красный bold.
- «Использование» — формат `0%`, подсветка по тем же порогам, что в таблице «По ролям» (зелёный/жёлтый/красный).
- «QA дефицит» считается как `min(0, available_qa - planned_qa)` — ноль или отрицательное.

- Текстовые ячейки lable: regular 10pt, выравнивание влево.
- Числовые ячейки: bold 11pt, выравнивание вправо, формат `#,##0`.
- Подсветка отрицательных значений (например, «QA дефицит −40») — красным жирным.

**Раздел «По ролям × месяцам»** (начинается через одну пустую строку, ~строка 8)
- Section-header `ПО РОЛЯМ × МЕСЯЦАМ`.
- Таблица. Колонки:

| Роль | Апр | Май | Июн | Σ Ёмкость | План | Остаток | % исп. |

- Месяцы — короткие подписи (Апр/Май/Июн для Q2 и т. д.). Брать из `QUARTER_MONTHS` и `MONTH_LABELS`.
- Месячная разбивка вычисляется через `ResourceBaseService.compute(scenario)` —
  `EmployeeBase.days[]` имеет `date` и `hours`, агрегируем по `(role, month)`.
- **Внешний QA**: если `scenario.external_qa_hours is not None`, в строке `qa` вместо посуточной суммы по штатным сотрудникам кладём `external_qa_hours`, поделённое на 3 месяца поровну (с округлением до 1 знака; остаток в последний месяц). `Σ Ёмкость` для QA = `external_qa_hours`.
- Σ Ёмкость = `available_by_role[role]` из `compute_summary`.
- План = распределение `planned_hours_by_role` (логика из прошлого спека: пропорция по `_demand_by_role` для каждой allocation с `included_flag=True`).
- Остаток = Ёмкость − План.
- % исп. = План / Ёмкость:
  - < 80 — фон `#DCFCE7`.
  - 80–110 — фон `#FEF3C7`.
  - > 110 — фон `#FEE2E2`, текст `#B91C1C` bold.
- Итоговая строка `ИТОГО` с суммой по столбцам, фон `#FAFAFA`, граница сверху 2pt тёмная.

**Раздел «По сотрудникам»** (через одну пустую строку)
- Section-header `ПО СОТРУДНИКАМ`.
- Таблица. Колонки:

| Сотрудник | Роль | Норма, ч | Отсутствия, ч | Доступно, ч | Дней отсутствия |

- Сортировка: команда → роль → имя.
- Норма = сумма календаря для сотрудника за квартал.
- Отсутствия = норма − доступно.
- Доступно = `EmployeeBase.total_hours`.
- Дней отсутствия = число рабочих дней с absence overlap.
- Цветовая подсветка «Дней отсутствия»:
  - 6–15 — жёлтый фон.
  - > 15 — красный фон.

### Заморозка / автофильтр / ширина колонок

- `freeze_panes = "A4"` — шапка-полоска и подзаголовок зафиксированы при прокрутке.
- Автофильтра на сводном листе нет (это не данные для фильтрации).
- Ширины: A — 26 (имя сотрудника), B — 18 (роль), C–H — по 12.

### Блок 1: Правила распределения часов на обязательные работы

- Section-header `1 · ПРАВИЛА РАСПРЕДЕЛЕНИЯ ЧАСОВ НА ОБЯЗАТЕЛЬНЫЕ РАБОТЫ`.
- Подзаголовок 9pt italic серый: «Сколько процентов нормы каждой роли резервируется на обязательные виды работ. Остаток уходит на инициативы.»
- Матрица:

- Колонки `wt*` — `MandatoryWorkType` с `subtracts_from_pool=True`, отсортированные по `sort_order`.

### Блок 3: Отсутствия команды в квартале

- Section-header `3 · ОТСУТСТВИЯ КОМАНДЫ В КВАРТАЛЕ`.

| Сотрудник | Роль | Причина | Тип | Начало | Конец | Дней | Часов |

  - > 80 — красный фон.

## Что НЕ делаем

1. **Графиков нет.** Только таблицы и условное форматирование. (Прежний спек не делал графиков, и редизайн тоже не делает.)
2. **Спарклайнов нет.** То же ограничение openpyxl, что и раньше.
3. **Сравнение сценариев** — отдельный endpoint в будущем, не здесь.
4. **Темы Excel.** Не используем темы и стили из шаблонов; всё пишем явно через `Font` / `PatternFill` / `Border`.
5. **PPTX-экспорт.** Не трогается. `ScenarioXlsxExporter` живёт независимо от `build_scenario_pptx`.
6. **Локализация.** Только русский.

## Архитектура реализации

`app/services/scenario_xlsx_export.py` (~990 строк сейчас) переписывается. Структура остаётся той же:

```python
class ScenarioXlsxExporter:
    def __init__(self, db, scenario_id): ...
    def build(self) -> bytes: ...
    def _load(self) -> ScenarioExportContext: ...
    def _sheet_summary(self, ws, ctx): ...     # бывший _sheet_cover
    def _sheet_included(self, ws, ctx): ...
    def _sheet_excluded(self, ws, ctx): ...
    def _sheet_reference(self, ws, ctx): ...    # объединяет правила + QA + отсутствия
```

Удаляются:
- `_sheet_by_role`
- `_sheet_by_employee`
- `_sheet_rules`
- `_sheet_absences`

Их логика частично переезжает: per-role × months → в `_sheet_summary`, per-employee → в `_sheet_summary` (упрощённая версия), правила/отсутствия → в `_sheet_reference`.

`_Style` упрощается — убираются все `*_BG` для cyan/accent, добавляется `DARK_HEADER = "1F2937"` (серый, не тёмно-синий).

`SHEET_NAMES` обновляется до 4 имён:
```python
SHEET_NAMES = ["Сводка", "Включено", "Не вошло", "Справочник"]
```

`ScenarioExportContext` остаётся почти таким же. Поле `period_end` (исключающее) сохранено, нужно для расчёта календарных границ.

Новый расчёт «по ролям × месяцам» добавляется в `_load` или вычисляется на лету в `_sheet_summary` из `resource_base.employees[].days`:

```python
def _per_role_per_month(ctx) -> dict[tuple[str, int], float]:
    """role -> month -> доступные часы (после вычета отсутствий)."""
    result: dict[tuple[str, int], float] = {}
    for emp in ctx.resource_base.employees:
        if not emp.role:
            continue
        for d in emp.days:
            key = (emp.role, d.date.month)
            result[key] = result.get(key, 0.0) + d.hours
    return result
```

`MONTH_LABELS` — короткие 3-буквенные («Янв», «Фев», …) — берутся из существующего `MONTH_LABELS` в `export_capacity_xlsx` (строки 380-383). Вытащить в общий модуль.
