# Точечное сохранение категории у одной задачи

**Дата:** 2026-06-11
**Страница:** `/categories` (`CategoriesEditorPage`)

## Проблема

Аналитик распределил задачу, позже понял — категория указана неверно. На вкладках кроме «К разбору» («Активные», «Инициативы», «Архив квартальных целей», «Архив неактуальных задач») у строки **нет** способа сохранить смену категории: выбор в выпадающем списке строки только стейджится в памяти и никуда не пишется.

Единственный путь — отметить галочку и нажать «Категория для отмеченных». Эта команда применяет категорию каскадом ко всему поддереву, поэтому поправить категорию **только у одной конкретной задачи** нельзя.

## Решение

Добавить точечную кнопку сохранения прямо в ячейку категории строки. Чисто фронтовая правка — backend уже поддерживает сохранение категории одной задаче без каскада (`verifyIssue` с `cascade=false` + `categoryCode`).

### Поведение

1. **Кнопка-галочка в ячейке категории.** Появляется справа от выпадающего списка строки **только когда** выбранная категория отличается от сохранённой (есть несохранённое изменение). Нет изменения — нет кнопки, таблица не засоряется. Работает одинаково на всех вкладках.

2. **Нажатие на листе (нет подчинённых):** сохраняет сразу, только эту задачу.

3. **Нажатие на родителе (есть подчинённые):** короткий вопрос (Popconfirm) с двумя вариантами:
   - «Только эту задачу» → каскада нет, подчинённые не тронуты (основной сценарий).
   - «И всё поддерево» → старое каскадное поведение для случая, когда оно нужно.

4. **После сохранения:** несохранённое изменение строки очищается, кнопка пропадает, дерево перезапрашивается (строка может уехать на другую вкладку, если сменилась категория).

### Что не меняется

- **Вкладка «К разбору»** — кнопка «Подтвердить» остаётся как есть (разбор стопки новых задач, каскадная отметка «разобрано»). Новая галочка-сохранение лишь добавляется поверх для строк, которые уже категоризованы и которые правят.
- **Команда «Категория для отмеченных»** — остаётся для массовых правок через галочки выделения. Новая кнопка закрывает точечный случай «одна задача».

## Техническая привязка

- Триггер сохранения: существующий `useVerifyIssue` → `verifyIssue(issueId, cascade, requireChildVerification=false, categoryCode, hasCategoryCode=true)`.
  - «Только эту» → `cascade=false`.
  - «И всё поддерево» → `cascade=true`.
- Кнопка показывается, когда `pendingCats.has(record.id)` **и** значение из `pendingCats` отличается от `record.assigned_category`.
- Очистка `pendingCats` для строки + `refreshLoadedChildren()` + инвалидация `['issues','tree']` — как в текущем `handleVerify`.
- Признак «есть подчинённые»: `record.has_children` (или загруженные `record.children`).

Новых эндпоинтов, миграций, изменений модели — нет.

## Критерии готовности

- На любой вкладке смена категории в строке показывает кнопку сохранения; без изменения кнопки нет.
- Сохранение листа меняет категорию только у него.
- Сохранение родителя через «Только эту задачу» не трогает подчинённых; через «И всё поддерево» — каскадит.
- После сохранения кнопка исчезает, дерево актуально.
- Вкладка «К разбору» и команда «Категория для отмеченных» работают как прежде.
