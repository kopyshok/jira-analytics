# Редизайн страницы «Сценарии» — устранение дублирования

**Дата:** 2026-04-22  
**Статус:** Approved

## Проблема

На странице «Сценарии» информация о ресурсе команды дублируется:
- Верхняя таблица (`ScenarioResourceSummary`) показывает разбивку по ролям: нормированные → обязательные работы → на бэклог
- Коллапс «Разбивка по ролям ↓» внутри правого блока (`PlanningCapacityPanel`) показывает ту же информацию (брутто → отпуска → обязательные → доступно)

Дополнительно: вкладки «Распределение / Правила» находятся внутри левой колонки, из-за чего правый блок начинается выше карточки бэклога — визуальное несоответствие.

## Решение

### 1. Убрать коллапс «Разбивка по ролям» из правого блока

В `PlanningCapacityPanel` компонент `<Collapse>` с `<ResourceBreakdownTable>` внутри карточки «Ресурс команды» — удаляется полностью.

Правый блок после изменения:
- Карточка «Ресурс команды» — только циферблат (большие цифры + прогресс-бар)
- Карточка «Ресурс по ролям» — бары план/доступно (без изменений)
- Карточка «По сотрудникам» — мини-бары по людям (без изменений)

### 2. Добавить блок «Отпуска» в верхнюю таблицу

`ScenarioResourceSummary` расширяется: справа от колонки «Итого» появляется отдельная секция «Отпуска квартала» с визуально отделённой рамкой (левая граница `2px solid border`).

Содержимое секции отпусков:
- Список сотрудников: цветная точка роли · имя · дни · вычитаемые часы
- Строка «Итого» — суммарные дни и часы
- Сотрудники без отпуска показываются как «—»

Данные берутся из уже существующего поля `summary.absence_days_by_employee`, которое сейчас рендерится только в удаляемом коллапсе.

### 3. Сделать верхнюю таблицу сворачиваемой

Добавить состояние `collapsed` (по умолчанию `false`).

**Развёрнуто:** полная таблица + блок отпусков, кнопка «↑ Свернуть»  
**Свёрнуто:** одна строка-бар с лейблом «На бэклог» и значениями по каждой роли + кнопка «↓ Развернуть»

Состояние сохраняется в `localStorage` под ключом `planning_resource_table_collapsed`, чтобы пользователь не разворачивал заново при каждом посещении.

### 4. Вынести вкладки над двуколоночный лейаут

Вкладки `<Tabs>` («Распределение» / «Правила») переносятся из левой колонки в `PlanningPage.tsx` — выше `<div style={{ display: 'grid', gridTemplateColumns: '...' }}>`.

Результат: оба блока (бэклог и правый блок) начинаются строго с одной горизонтальной линии.

## Что не меняется

- Визуальный дизайн верхней таблицы: цвета ролей, разделители колонок, стиль строк
- `BacklogRoleCell` (цветные ячейки АН/ПР/ТС/ОПЭ в строках бэклога)
- Логика расчётов и API — только перестановка UI
- Компонент `ResourceBreakdownTable` — удаляется вместе с коллапсом

~3–4 часа. Только frontend, без бэкенда и миграций.
