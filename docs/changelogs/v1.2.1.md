# v1.2.1 — Что нового

## Новое

### Бэклог

**Длинная RFA: остаток часов по ролям и кварталам**

У RFA с дочерними Эпиками появилось раскрытие строки в бэклоге. Внутри — таблица по ролям (Аналитик / Разработка / Тестирование / ОПЭ) с шестью колонками: План, Факт прошлых кварталов, Факт текущий, Утверждено, Запланировать, Черновик. Период текущего квартала берётся из глобального фильтра сверху страницы.

**Режим планирования группы RFA**

В раскрытой строке RFA — переключатель «RFA целиком / По Эпикам». В режиме «По Эпикам» вы отмечаете отдельные дочерние Эпики; RFA-родитель остаётся доступным как опциональная галочка для непокрытых кварталов. Пересечение «RFA + дочка в одном квартале» подсвечивается оранжевым.

**Ручная правка плановых часов с историей**

Кнопка «Редактировать план» в раскрытой RFA или у любой задачи открывает drawer: вводите часы по ролям, обязательный комментарий — и план обновляется без обхода через Jira. История всех правок (кто, когда, что) хранится и доступна по ссылке «Показать историю». Если в Jira план потом изменили — появится баннер «Принять Jira / Игнорировать», ваша правка не теряется автоматически.

### Сценарии

**Разбивка часов RFA в сценарии**

Рядом с ключом длинной RFA в сценарии появилась иконка ℹ. Клик — drawer с таблицей часов (План / Факт прошлых Q / Факт текущий / Утверждено / Запланировать / Черновик per роль) для выбранного квартала.

### Категоризация задач

**Прыжок к задаче по Jira-ключу (β)**

Введите ключ задачи в поле поиска на странице «Категоризация задач» и нажмите Enter — дерево автоматически раскроет путь от корня до найденной задачи, проскроллит к ней и подсветит жёлтым мерцанием. Дерево больше не фильтруется — соседние ветки остаются видны для сравнения. Режим включён по умолчанию.

## Улучшения

### Бэклог

**Параметры планирования — 2 колонки**

Окно «Параметры планирования» для RFA-задачи теперь раскладывает Анализ/Разработку слева, Тестирование/ОПЭ справа. Скролл вниз сократился примерно вдвое.

### Категоризация задач

**«Применить» теперь каскадно проставляет категорию потомкам**

При назначении категории родителю через «Применить» все потомки без своей категории получают ту же категорию. Если у потомка стоит другая категория, его поддерево не трогается — ручные решения сохраняются. В уведомлении после применения видно, сколько потомков унаследовало категорию.

### Общее

**Справка обновлена под изменения RFA+эпиков и категорий**

Разделы «Бэклог», «Категоризация задач» и «Планирование сценариев» приведены к актуальному виду: переименование кнопки «Обновить с Jira», переезд таблицы часов в модалку «Параметры планирования», отказ от перетаскивания в Бэклоге, описание каскадного применения категорий, кнопки «Подтвердить» (категория + verify за один шаг) и режима поиска-прыжка по Jira-ключу. Удалены упоминания убранных элементов («Верифиц. детей», полоса черновика, «Массовые операции»).

## Исправления

### Общее

**Фильтр команд в шапке стал удобнее**

Список команд переделан: добавлен поиск, чек-лист с подсветкой выбранных и переключатель «выбрать все / найденные». Кнопка «Применить» больше не перекрывается выпадающим списком.

### Бэклог

**Раскрытие RFA: убраны три предупреждения в консоли**

На вкладке «Активные» при раскрытии RFA-строки в консоли больше не сыпятся предупреждения AntD 6 о вертикальной компоновке кнопок режима и вложенной таблице часов.

**Кнопка «Обновить с Jira» теперь реально ходит в Jira**

Раньше кнопка лишь перечитывала локальную базу и показывала «Данные обновлены», но не подтягивала свежие значения полей задач (Заказчик, Тип затрат, Исполнитель). Теперь нажатие действительно тянет эти поля из Jira для всех связанных задач бэклога и обновляет их в карточках. Кнопка переименована в «Обновить с Jira» и показывает индикатор загрузки во время запроса.

**Отменённые задачи больше не считаются в «Черновике» длинной RFA**

В разделе «Целевые задачи» → «Параметры планирования» колонка «Черновик» больше не включает потомков в статусе «Отменено» / «Rejected» / «Won't Do». Раньше отменённый Эпик или ITL без ворклога подмешивал свои плановые часы в Черновик.

### Категоризация задач

**«Подтвердить» применяет категорию и сразу убирает задачу со стека**

Раньше после выбора категории в выпадайке и нажатия «Сохранить изменения» задача оставалась в «К разбору» — приходилось отдельно жать «Подтвердить» в строке. Теперь шагов один: выбрать категорию в Select родителя → нажать «Подтвердить +N» в той же строке. Категория проставляется родителю + автоматически наследуется невериф потомкам (уже разобранные задачи с собственной категорией не меняются). Кнопка «Сохранить изменения» внизу больше не нужна и удалена. Bulk-модалка «Категория для отмеченных» тоже применяет + подтверждает сразу. Кнопка «Массовые операции» убрана.

**Чужие эпики на /categories теперь видно как контекст + умный поиск**

Раньше задачи под чужим эпиком (где родитель — другой команды) на /categories пропадали полностью из дерева — PM их не находил. Теперь чужой эпик подтягивается как серый read-only якорь («контекст»): PM видит в какой ветке висит задача, может работать со своими дочками, но сам чужой эпик править нельзя — он принадлежит другой команде. Та же логика для всех вкладок: К разбору, Активные, Архивы. Поиск по Jira-ключу стал строгим: «ITL-57» теперь матчит ровно «ITL-57» — раньше выводил всё подряд (ITL-571, ITL-579 и т.д.). Текстовый поиск по названию работает как раньше. Найденная строка пульсирует жёлтой подсветкой.

**«Подтвердить» в стеке: каскад добивает уже-проверенных потомков без своей категории**

Если у потомков в подветке нет своей категории, но они когда-то были помечены как «проверенные» (например, после старого пересчёта), кнопка «Подтвердить +N» больше не пропускает их — родительская категория проставляется и им. Граница каскада не менялась: задача с ручной категорией остаётся со своей.
