# Отчёт по видам работ

Раздел `/analytics/work-type-report` — тематический отчёт. Берёт фактические трудозатраты за выбранный период и показывает, на какие **направления работы** они ушли. Темы определяет ИИ автоматически из названий и описаний задач, не из жёсткого справочника.

---

## 1. Зачем раздел

Аналитика и Dashboard показывают «сколько часов на проект / на сотрудника / на категорию». Этот раздел отвечает на другой вопрос: **«какие тематические направления съели время команды в этом квартале»**. Не «логин и регистрация» как Jira-проект, а «доделки авторизации», «поддержка релиза», «миграция на новую платформу» — содержательные темы, которые сами выкристаллизовались из задач.

Применение:
- Готовим квартальный отчёт руководителю — нужна одна страница «куда ушёл квартал».
- Подозреваем, что много часов уходит на «невидимое сопровождение», а не на квартальные цели — хотим увидеть это явно.
- Хотим сравнить, сколько направление получило в одном квартале vs в другом.
- Нужно понять, есть ли «пожарные темы», которые незаметно растут.

Этот отчёт **не** заменяет Аналитику. Аналитика — детальный иерархический отчёт «проект → эпик → задача → ворклог». Тематический отчёт — другой угол: «темы → задачи в теме». Один не заменяет другой, они дополняют друг друга.

---

## 2. Когда сюда заходить

- Перед квартальным/полугодовым отчётом руководству — взять одну страницу с темами и AI-сводкой.
- Когда непонятно, на что ушло время команды, а в Аналитике видно только «много мелких задач».
- После большой синхронизации, чтобы посмотреть, появились ли новые темы.
- Когда нужен PDF — отдельная страница «PDF для руководства» открывается в новой вкладке и печатается через браузер.
- Раз в квартал — пересчитать темы (ИИ может найти новые направления).

## 3. Элементы интерфейса

| № | Элемент | Что показывает |
|---|---------|----------------|
| 1 | Заголовок «АНАЛИТИКА / Тематический отчёт» | Локация в навигации |
| 2 | Период (в шапке приложения) | Год + квартал + опционально месяц — общий пикер для всей аналитики |
| 3 | Фильтр команды (в шапке приложения) | Глобальный, режет, какие задачи и какие сотрудники попадают в отчёт |
| 4 | Селектор вида работы + кнопка «Применить» | Переключение между видами работы (Аналитика, Разработка, Тестирование и т. д.) |
| 5 | Плашка свежести (Свежий / Устарел / Нет данных) | Когда отчёт был построен; «Устарел» появляется, если после построения меняли словарь тем |
| 6 | Кнопка «Словарь тем» | Открывает справочник тем — активные, архив, кандидаты на добавление |
| 7 | Кнопки «Пересчитать», «XLSX», «PDF для руководства» | Управление построением и выгрузка |
| 8 | AI-сводка (заголовок) | Одна фраза от ИИ — главный вывод по отчёту |
| 9 | Плитки KPI (Часов / Тем / Задач / Сотрудников) | Тоталы по выбранному периоду + команде + виду работы |
| 10 | Бублик «Распределение тем» | Топ-5 тем + «Другое»; клик по сегменту подсвечивает тему в таблице |
| 11 | Карточка «Рекомендация» | ИИ-рекомендация + ожидаемый эффект (если есть) |
| 12 | Панель «Кандидаты» | Темы-кандидаты, которые ИИ предложил, но они не приняты в словарь |
| 13 | Управление группировкой | Пресеты + drag-чипы + сохранённые макеты |
| 14 | Таблица «Темы → задачи» | Иерархия по выбранной группировке; раскрытие темы показывает задачи |
| 15 | Блок «Требуют ручной классификации» | Задачи, которые ИИ не смог уверенно отнести ни к одной теме |
| 16 | Кнопка «Справка» (в этом окне) | Открывает эту страницу |

Подробнее по ключевым блокам — ниже.

### 3.1. Селектор вида работы

«Вид работы» — это какая роль смотрит отчёт: Аналитика, Разработка, Тестирование, ОПЭ и т. д. Каждый вид работы имеет свой словарь тем (у аналитиков и разработчиков могут быть разные темы) и считает часы только по своей роли. Список видов задаётся в Настройках → Виды работ.

Выбрать вид → «Применить». Выбор запоминается между сессиями.

### 3.2. Плашка свежести

- **«Свежий»** — отчёт построен по текущему словарю тем.
- **«Устарел (словарь обновлён)»** — после построения отчёта в словарь добавили/изменили/архивировали тему. Цифры показываются как есть, но в них нет последних правок словаря — нужно нажать «Пересчитать».
- **«Нет данных»** — отчёт ещё не строили; см. блок «Тематический отчёт ещё не построен» в центре экрана.

Под плашкой во всплывающей подсказке — дата и время последнего построения.

### 3.3. AI-сводка

Карточка с бирюзовой полосой слева, шрифт крупный. Это **главная фраза** от ИИ по выбранному периоду — обычно один-два предложения: «Команда в основном занималась X и Y; Z потребовала больше часов, чем планировалось». Под фразой — служебная строка: модель ИИ, цифры тоталов, версия промпта, плашка «AI» или «шаблон».

«Шаблон» — значит, ИИ ответ не дал (выключен или сломан), и текст собран автоматически из тоталов. Это не ошибка, но и не «настоящая» сводка.

### 3.4. Плитки KPI

Четыре числа: часов за период, сколько разных тем нашлось, сколько задач попало в отчёт, сколько сотрудников логировали. Все цифры — после применения глобального фильтра команды и периода.

### 3.5. Распределение тем (бублик)

Бублик показывает топ-5 тем по часам + остальное группируется в «Другое». В центре — общий объём часов. Справа — те же темы списком с полоской прогресса. Клик по сегменту бублика или по полоске **подсвечивает** соответствующую тему в таблице ниже.

### 3.6. Рекомендация и кандидаты

- **Рекомендация** — карточка с жёлтой полосой слева. Что ИИ предлагает по результатам отчёта (например, «обратите внимание на тему X — она забирает часов больше нормы»). Может отсутствовать, если ИИ нечего сказать.
- **Кандидаты** — темы, которые ИИ нашёл во время разбора, но **не** включил в основной словарь (неуверенный или редкий случай). Видны мини-карточками. Кнопка «Просмотреть всех» открывает «Словарь тем» на вкладке «Кандидаты», где можно принять, объединить с существующей или проигнорировать.

### 3.7. Управление группировкой

Сверху — пять пресетов: «По темам / По сотрудникам / По командам / По ролям / По проектам». Активный пресет подсвечен.

Под пресетами — строка «Порядок:» с чипами. Чипы можно перетаскивать (меняет порядок группировки), удалять крестиком, добавлять плюсиком из выпадающего списка. Минимум один чип — таблица всегда хотя бы по чему-то группируется.

Справа — «Сохранить как…» — называете свой макет и при желании ставите его «по умолчанию» (звёздочка). Сохранённые макеты показываются отдельной строкой, удаляются крестиком.

Макет по умолчанию автоматически применяется при открытии раздела для этого вида работы.

### 3.8. Таблица «Темы → задачи»

Главная сводная таблица. Строки бывают трёх типов:
- **Строка темы** — название темы + цветной маркер + часы + % от общего + число задач + число сотрудников. Раскрытие темы показывает её задачи (или промежуточный уровень группировки, если в порядке есть Команда/Роль/Сотрудник/Проект перед «Задачей»).
- **Группа** (если в порядке есть Команда / Роль / Сотрудник / Проект) — промежуточная строка-заголовок. Без отдельной AI-сводки, только цифры.
- **Строка задачи** — ключ Jira, название, часы по сотруднику и теме. Клик по строке открывает «детальную карточку задачи» (drill-down справа).

У темы могут быть метки:
- **«Новая»** — тема впервые появилась в этом построении (раньше ИИ её не выделял).
- **«Низкая уверенность»** — ИИ нашёл тему, но мало задач или неоднозначные совпадения.

Под названием темы (после раскрытия) — короткий нарратив от ИИ + список «опорных» ключей задач, по которым ИИ узнал тему.

### 3.9. Блок «Требуют ручной классификации»

Снизу таблицы. Задачи, которые ИИ **не** смог уверенно отнести ни к одной теме. По каждой строке — селектор «Выбрать тему» и кнопка «Назначить». Назначенная вручную тема запоминается и применится при следующем построении. Используйте, чтобы дотянуть отчёт до 100% покрытия.

### 3.10. Печать в PDF

Кнопка «PDF для руководства» открывает в новой вкладке **отдельную страницу** в светлой теме, оптимизированную под печать (A4, без сайдбара). В верхней правой части — кнопка «Печать» — вызывает диалог печати браузера. В диалоге выберите «Сохранить как PDF» (или физический принтер).

Печатная страница содержит: шапку (вид работы, период, команда), AI-сводку, распределение тем, топ-5 тем (с нарративом и топ-3 задач), рекомендацию. Колонтитулы, поля и масштаб настраиваются в диалоге печати браузера — это особенность браузерной печати, а не сервиса.

## 4. Откуда цифры

- **Темы** — ИИ распознаёт направления из названий и описаний задач. Сначала каждое название/описание превращается во встроенное представление (embedding). Затем задачи группируются по пересечению **тематических маркеров** — устойчивых фраз, которые часто встречаются вместе. Темы попадают в **словарь** — он переиспользуется между построениями и доучивается со временем (см. «Словарь тем»).
- **Часы по теме** — фактический worklog из Jira по задачам этой темы, отфильтрованный глобальным фильтром команды. Учитываются только сотрудники из выбранных команд.
- **% от общего** — часы темы / часы по всему отчёту (после фильтра команды и периода).
- **Сотрудники в теме** — кто залогировал часы хотя бы в одну задачу этой темы.
- **AI-сводка / рекомендация / нарратив темы** — текст генерирует языковая модель. Кэшируется по комбинации (вид работы, период, команда, версия промпта, словарь). Если меняется любая из этих составляющих — кэш не подходит, нужен пересчёт.
- **Период** — берётся из общего пикера в шапке. Часы фильтруются по дате ворклога: попадают только записи внутри квартала/месяца.
- **Команда** — глобальный фильтр в шапке. Управляет, чьи ворклоги учитывать.
- **Вид работы** — отдельный словарь тем. Часы суммируются только по тем сотрудникам, которые исполняют выбранную роль на задаче (например, «Аналитик» = тот, кто залогировал часы как аналитик).

«Шаблон» вместо AI-сводки появляется, когда ИИ выключен глобально (kill switch в Настройках → ИИ) либо когда модель ответила ошибкой.

## 5. Типовые задачи

### Сценарий А: подготовить квартальный отчёт руководителю

1. В шапке выбрать нужный год + квартал.
2. В шапке выбрать команды, которые входят в отчёт (или все ваши команды).
3. Открыть «Отчёт по видам работ», выбрать нужный вид работы → «Применить».
4. Если плашка свежести — «Устарел» или «Нет данных», нажать «Пересчитать» (или «Построить первый отчёт»). Дождаться окончания (всплывает прогресс-окно).
5. Проверить AI-сводку наверху — это будет ваш «главный тезис».
6. Посмотреть на бублик — топ-5 тем + «Другое».
7. Нажать «PDF для руководства» → в новой вкладке нажать «Печать» → «Сохранить как PDF».

### Сценарий Б: уточнить, почему направление получило мало часов

1. В отчёте найти интересующую тему в таблице.
2. Раскрыть её — видны задачи внутри + нарратив ИИ.
3. Перетащить чип «Сотрудник» в порядок группировки (или нажать пресет «По сотрудникам»). Теперь видно, кто именно работал по теме и сколько.
4. Если задач мало или часы малы — возможно, тема либо неточная (ИИ зацепил несколько случайных совпадений), либо команда правда не уделяла ей время.
5. Кликнуть на отдельную задачу — открывается детальная карточка с историей ворклога.

### Сценарий В: добавить новое направление в словарь тем

1. В разделе посмотреть панель «Кандидаты» справа — это темы, которые ИИ нашёл, но не принял.
2. Нажать «Просмотреть всех кандидатов» → открывается «Словарь тем» на вкладке «Кандидаты».
3. Если кандидат подходит — «Принять» (станет полноценной темой) или «Объединить с…» (если это вариация существующей темы).
4. Если кандидат шум — «Игнорировать».
5. Вернуться в отчёт — плашка свежести стала «Устарел». Нажать «Пересчитать», чтобы новый кандидат вошёл в распределение.

### Сценарий Г: вручную классифицировать задачи, которые ИИ не разобрал

1. Прокрутить отчёт до блока «Требуют ручной классификации» внизу.
2. У каждой строки — селектор тем. Выбрать подходящую тему → «Назначить».
3. Назначения сохраняются и применятся в **следующем** построении отчёта. Текущий снимок не меняется автоматически — для отражения нажать «Пересчитать».

## 6. Ограничения

- **Отчёт строится в фоне** — на больших командах и больших периодах построение занимает несколько минут. Прогресс показывается в модальном окне. Закрывать вкладку нельзя — построение прервётся.
- **ИИ может ошибаться** — особенно в темах с пометкой «Низкая уверенность». Доверяйте AI-сводке как первой версии, не как истине в последней инстанции; проверяйте по топ-задачам внутри темы.
- **Кэш AI-текста** — если изменился промпт (версия промпта в служебной строке) или словарь тем, нужен пересчёт. Без пересчёта AI-сводка может «отставать».
- **«Темы» — не «категории»** — это разные сущности. Тема живёт внутри вида работы и определяется ИИ из текста задачи. Категория ставится вручную в разделе «Категоризация задач» и определяет, что задача вообще учитывается в фактах.
- **Печатная страница использует светлую тему и оптимизирована под A4** — на других форматах может «поплыть» вёрстка; настраивайте поля в диалоге печати браузера.
- **XLSX-выгрузка** содержит данные тоталов и тем, но **не** содержит AI-нарратив постранично — это технический срез, не презентация.
- **«Применить» для вида работы** — без него выбор не сохранится. Это намеренно: чтобы не пересчитывать на каждый щелчок селектора.

## 7. Частые проблемы

| Симптом | Причина | Что делать |
|---------|---------|------------|
| Весь раздел заблокирован сообщением «выключен администратором» | ИИ выключен глобально (kill switch в Настройках → ИИ) | Обратиться к админу; без ИИ раздел не работает в принципе |
| AI-сводка показывает «шаблон» / «AI-сводка появится после первого расчёта» | ИИ ответил ошибкой, либо отчёт не строился, либо нет квоты у модели | Нажать «Пересчитать»; если повторяется — проверить в Настройках → ИИ статус выбранного провайдера |
| Задача попала не в ту тему | ИИ ошибся на похожих формулировках | Открыть «Словарь тем», у нужной темы добавить алиас под формулировку задачи; либо в Настройках убрать тему-конкурент в архив. Затем — «Пересчитать». Для разовых случаев — присвоить вручную через блок «Требуют ручной классификации» |
| Задача вообще не попала ни в одну тему | ИИ не нашёл устойчивых маркеров (короткое название, шум) | Блок «Требуют ручной классификации» внизу — выбрать тему вручную |
| Цифры не совпадают с Аналитикой | Разный фильтр периода / команды / вида работы; либо ИИ исключил задачу из распределения; либо отчёт устарел и не пересчитан | Проверить пикер периода и фильтр команды в шапке; убедиться, что вид работы соответствует роли сотрудников; если плашка «Устарел» — нажать «Пересчитать» |
| Плашка «Устарел» не уходит после пересчёта | После пересчёта снова изменили словарь тем | Это нормально — каждое изменение словаря помечает отчёт как устаревший; пересчитывайте после того, как словарь приведён в порядок |
| Печатная страница криво ложится в PDF (часть текста уехала, поля огромные) | Браузерные настройки печати: масштаб, поля, колонтитулы | В диалоге печати → «Параметры» → выставить «Поля: минимальные», «Масштаб: по размеру», убрать колонтитулы. Это особенность браузера, не сервиса |
| Бублик показывает «Темы ещё не приняты» | Словарь пуст или все темы в архиве; только кандидаты | Открыть «Словарь тем» → «Кандидаты», принять подходящие — диаграмма появится после пересчёта |
| Кнопка «Пересчитать» висит без прогресса | Идёт длинный шаг (embedding или LLM-запрос); либо ошибка соединения | Дождаться или закрыть окно прогресса и попробовать снова |

## 8. Связанные разделы

**Откуда приходят данные:**
- **Синхронизация** (`/sync`) — без свежей синхронизации задач и ворклогов отчёт работает на старых данных.
- **Категоризация задач** (`/categories`) — если у задач не проставлена категория, они **не** попадают в факты и, соответственно, в темы. Чистая категоризация = чистый тематический отчёт.
- **Настройки → Виды работ** — определяют, какие виды работы доступны в селекторе.
- **Настройки → Категории** — определяют, какие категории идут в факты.
- **Настройки → ИИ** — провайдер + модель + kill switch. Без включённого ИИ раздел недоступен.
- **Глобальный фильтр команды** (в шапке) — режет, чьи ворклоги учитывать.
- **Общий пикер периода** (в шапке) — год / квартал / опц. месяц.

**Куда уходит результат:**
- **Печатная страница** (`/analytics/work-type-report/print`) — открывается в новой вкладке для PDF.
- **XLSX-выгрузка** — кнопка «XLSX» в шапке отчёта.

**Соседние отчёты:**
- **Аналитика** (`/analytics`) — детальный иерархический отчёт «проект → эпик → задача → ворклог». Здесь — тематический срез поверх тех же фактов.
- **Dashboard** (`/`) — KPI команды.
- **Executive Dashboard** (`/executive`) — верхнеуровневые KPI и AI-сводка по всему сервису.
