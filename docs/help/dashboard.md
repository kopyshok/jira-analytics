# Главная (Dashboard)

Раздел `/` — обзорная страница, которую открывает каждый сотрудник в начале дня. Три виджета на одном экране показывают, что сейчас происходит у команды: какие проекты квартала живут, кто перегружен или простаивает, на что фактически уходят часы. Все цифры на странице — это срез фактов и плана на выбранный период с учётом глобального фильтра команды из шапки.

---

## 1. Зачем раздел

Главная — это «приборная панель аналитика»: за 30 секунд понятно, нужны ли срочные действия. Не нужно переключаться между Аналитикой, Capacity и Проектами — три виджета покрывают три ключевых вопроса:

- **Что с проектами квартала?** — какие закроются в срок, какие просрочены, какие молчат.
- **Кто перегружен, кто простаивает?** — план/факт по каждому сотруднику в разрезе нормированных работ.
- **На что уходит время команды?** — раскладка часов по категориям задач + последняя активность каждого сотрудника.

Если на дашборде всё ровно — можно идти заниматься своей текущей задачей. Если что-то светится красным — кликом проваливаешься в нужный раздел и разбираешься.

---

## 2. Когда сюда заходить

- В начале рабочего дня — «утренний обзор» (5 минут).
- Перед статусом руководству — снять KPI по проектам и загрузке.
- Когда кто-то из команды жалуется на перегруз — проверить второй виджет.
- Когда руководство спрашивает «куда уходят часы» — третий виджет показывает раскладку.
- После большой синхронизации — убедиться, что цифры обновились и нигде нет нулей.

---

## 3. Элементы интерфейса

В шапке приложения — два глобальных переключателя, которые управляют всеми тремя виджетами:

- **Период** (год + квартал + опционально месяц).
- **Фильтр команды** (одна или несколько команд).

Меняешь в шапке — все три виджета пересчитываются.

Каждый виджет — отдельная карточка во всю ширину. У большинства виджетов в правом верхнем углу есть шестерёнка с настройками вида (какие колонки/блоки/пороги показывать). Настройки сохраняются в браузере для каждого сотрудника отдельно.

### 3.1. Виджет «Проекты квартала»

Верхний виджет. Показывает все проекты, входящие в утверждённый сценарий выбранного квартала, и их состояние.

Состоит из четырёх блоков (часть можно отключить шестерёнкой):

- **Круговая диаграмма** слева — сводка по статусам проектов: выполнены, в работе, просрочены, не начаты. В центре общее число проектов.
- **Таблица проектов** в центре — по одной строке на проект. Колонки настраиваются:
  - **Статус** — текущий статус из Jira.
  - **Задачи** — выполнено / всего подзадач.
  - **Команда** — аватарки исполнителей.
  - **Срок** — дата дедлайна и сколько дней осталось (красным — если просрочен).
  - **Тренд** — стрелка вверх/вниз и сколько часов залогировано за прошлую неделю.
  - **Прогноз** — когда проект закроется по текущему темпу; восклицательный знак — если не успевает в квартал.
  - **Прогресс** — горизонтальная полоса загрузки.
  - **Факт / План** — часы фактом и план.
  - **%** — отношение факта к плану (>110% красный, 70–110% зелёный, ниже жёлтый).
- **KPI-плитки** справа — четыре крупных показателя: всего часов фактом, средняя загрузка квартала, сколько проектов молчат больше 14 дней, сколько закроются в срок по прогнозу.
- **Активность по неделям** — спарклайны топ-6 проектов по фактическим часам.

Рядом с названием проекта могут появиться плашки: «тишина Nд» — давно никто не логировал; «+N ч» — перерасход относительно плана.

Клик по строке проекта → открывается карточка проекта в разделе «Проекты».

### 3.2. Виджет «Нормированные работы»

Средний виджет. Показывает план/факт по каждому сотруднику команды, сгруппированный по роли (Аналитик / Разработка / Тестирование / ОПЭ).

Каждая колонка — это роль. Внутри роли — её сводка (план / факт / средняя загрузка) и список сотрудников. У каждого сотрудника:

- Аватарка, имя, итоговый процент загрузки (красный — перегруз, жёлтый — норма, зелёный — есть запас).
- Полоса «план / факт». Если перерасход — красный «хвост» уходит за отметку плана.
- Подпись «факт Nч · план Mч».
- Разбивка по видам работ внутри роли — мини-полоски и цифры «факт / план» по каждому виду.
- Если у сотрудника много часов на «чужие» команды (≥8 ч или ≥5%) — рядом с именем алая плашка «чужие N ч · M%».

Шестерёнка в правом верхнем углу виджета — настройка порогов «перегруз/недозагрузка» в процентах (по умолчанию >110% перегруз, 70–110% норма, <70% запас).

Клики:
- По имени сотрудника → Аналитика, отфильтрованная на него.
- По строке вида работ → Аналитика, отфильтрованная на него + конкретный вид работ.
- По иконке диаграммы справа от вида работ → Тематический отчёт по этому виду работ.

### 3.3. Виджет «Ворклоги по категориям задач»

Нижний виджет. Делится на две части.

**Часть А — раскладка часов по категориям** (слева). Сетка плиток 5 в ряд — топ-10 категорий по количеству часов. На каждой плитке:

- Название категории и её процент от общего объёма (плашка справа сверху).
- Крупная цифра — часы.
- Подпись «N wl · M зад · K чел» — сколько ворклогов, задач и сотрудников вовлечено.
- Полоска внизу — визуальный процент.

Если категорий больше 10 — справа внизу появляется плитка «+ ещё N» с остатком часов.

Справа от плиток — таблица с теми же данными в строчном виде (категория, часы, ворклоги, задачи, сотрудники, средняя длина ворклога в минутах, %), плюс итоговая строка.

Под плитками — сводная полоса: общее количество часов, ворклогов, задач, категорий и средняя длина ворклога.

Клик по плитке категории → Аналитика, отфильтрованная по этой категории.

**Часть Б — последний ворклог сотрудника** (внизу виджета). По карточке на каждого сотрудника команды: аватарка, имя, плашка «отсутствует / больничный / отпуск» если сейчас в отсутствии, и цветной таймер «N дней назад». Цвет определяется порогами:

- зелёный — недавно (по умолчанию ≤3 дней);
- жёлтый — задерживается (по умолчанию 4–5 дней);
- красный — давно (всё, что больше).

Пороги настраиваются шестерёнкой в правом верхнем углу виджета.

Клик по карточке сотрудника → Аналитика, отфильтрованная на него.

## 4. Откуда цифры

Все три виджета считаются на одних и тех же данных:

- **Источник всех часов** — ворклоги из Jira, привязанные к задачам с проставленной категорией. Без категории задача в дашборд не попадёт — она висит в очереди «К разбору» в разделе «Категоризация».
- **Источник проектов** — задачи-проекты из утверждённого сценария выбранного квартала. Если сценарий не утверждён или пуст — первый виджет покажет «Нет проектов в утверждённом сценарии квартала».
- **Источник плана** — план по нормированным работам, рассчитанный из базы ресурсов сотрудников (рабочий календарь минус отсутствия) и правил распределения времени по ролям/видам работ.
- **Категории** — берутся из общего справочника. Системные категории (Архив, Инициативы, Незаполненные worklog) считаются по обычным правилам, но архивные и инициативы не считаются «фактом работы».
- **Фильтр команды** — глобальный, из шапки. Влияет на всё: какие сотрудники попадают во второй виджет, чьи ворклоги учитываются в часах, какие проекты считаются «своими».
- **Период** — тоже из шапки. Можно смотреть весь квартал или один месяц внутри квартала.

«Чужие часы» в виджете нормированных работ — это часы, которые сотрудник твоей команды залогировал на задачи **другой** продуктовой команды. Они уже включены в общий факт, плашка просто отдельно подсвечивает.

## 5. Типовые задачи

### Сценарий А: утренний обзор (5 минут)

1. Открыть `/`.
2. Проверить, что в шапке стоит твой период и твоя команда.
3. Посмотреть в KPI-плитки первого виджета: средняя загрузка и сколько проектов закроются в срок.
4. Глазами пройти по таблице проектов — обратить внимание на красные плашки «тишина Nд» и «+N ч перерасход».
5. Во втором виджете — отметить, есть ли сотрудники с процентом красным (перегруз) или жёлтым.
6. В третьем виджете — глянуть карточки «последний ворклог»: есть ли кто-то в красной зоне (давно не логировал).
7. Если всё ровно — закрыть. Если что-то светится — провалиться кликом в нужный раздел.

### Сценарий Б: проверка перегруза сотрудника

1. На дашборде во втором виджете видишь у сотрудника красный процент (>110%).
2. Клик по имени → открылась Аналитика, отфильтрованная на него.
3. Видишь, на что именно ушли часы.
4. Если перерасход на «чужой» команде — это видно ещё на дашборде по плашке «чужие N ч».
5. Дальше — решение организационное (перераспределить нагрузку / пересогласовать план / обсудить с сотрудником).

### Сценарий В: проваливание в проект

1. На дашборде в первом виджете видишь у проекта красную плашку «+N ч» или «тишина Nд».
2. Клик по строке проекта → открылась карточка проекта в разделе «Проекты».
3. Там видно, какие задачи перерасходовали или давно не двигались.

### Сценарий Г: на что уходят часы команды

1. Открыть `/`.
2. Промотать к третьему виджету.
3. Самые крупные плитки сверху-слева — это категории, на которые ушло больше всего часов.
4. Клик по плитке нужной категории → Аналитика, отфильтрованная по этой категории. Видно, какие сотрудники и задачи в ней.

### Сценарий Д: подготовка статуса руководству

1. Зафиксировать четыре KPI с первого виджета: всего часов / средняя загрузка / молчат >14 д / закроются в срок.
2. Во втором виджете — отметить итоговую загрузку по ролям.
3. В третьем виджете — взять топ-3 категории по часам.

## 6. Ограничения

- **Нет своего фильтра в виджетах** — фильтр команды и период живут только в шапке. Нельзя одновременно смотреть две команды раздельно — только их объединение.
- **Настройки колонок и порогов** — сохраняются в браузере, не синхронизируются между устройствами. На другом компьютере увидишь дефолтные значения.
- **«Проекты квартала» работает только при утверждённом сценарии** — если сценарий в черновике или его нет, виджет покажет «Нет проектов».
- **Прогноз «закроется в срок»** — это экстраполяция текущего темпа логирования. Если темп изменится — изменится и прогноз.
- **«Тишина Nд»** — считается по последнему ворклогу на любой задаче проекта. Если кто-то логировал, а потом откатил — может ложно показывать активность.
- **«Чужие часы»** — границы «свой/чужой» определяются глобальным фильтром команды. Если фильтр не выбран — плашки чужих часов не будет.
- **Категории «Архив» и «Инициативы (на потом)»** — не считаются как факт работы во втором виджете, но видны в третьем (раскладка по категориям) для прозрачности.

## 7. Частые проблемы

| Симптом | Причина | Что делать |
|---------|---------|------------|
| Дашборд пустой / «Нет данных» во всех виджетах | Фильтр команды в шапке отсекает всех сотрудников и все задачи | Проверить шапку: выбрана ли команда, не выбран ли пустой/несуществующий набор команд |
| В виджете «Проекты квартала» — «Нет проектов в утверждённом сценарии» | Сценарий квартала не утверждён или пуст | Раздел «Планирование сценариев» → утвердить сценарий с проектами |
| Везде нулевые часы фактом | Ворклоги ещё не загружены из Jira | Раздел «Синхронизация» → запустить полный синк или backfill ворклогов с нужной даты |
| Часы в категориях есть, а во втором виджете план/факт ровные нули | Не настроена база ресурсов или правила распределения времени по ролям | Раздел «Capacity» → проверить правила и базу ресурсов; раздел «Категоризация» → проверить, что задачам проставлены категории |
| Цифры на дашборде не сходятся с Аналитикой по той же команде/периоду | Разница в округлении (дашборд округляет до целых часов, Аналитика показывает с десятыми) или разные фильтры внутри Аналитики | Сравнивать на одинаковом фильтре, держать в уме округление |
| У сотрудника красная плашка «чужие N ч», но в Аналитике этих часов не видно | Аналитика отфильтрована на «свою» команду и эти часы (на чужие проекты) попадают в другую ветку отчёта | Снять фильтр команды в Аналитике или переключиться на отчёт по сотруднику |
| Прогноз показывает «закроется к Nд после квартала» (с восклицательным знаком), но проект кажется в порядке | Темп логирования последних недель ниже, чем нужно для закрытия в срок | Проверить тренд в той же строке; если тренд вверх — прогноз скоро улучшится сам, если флэт/вниз — реальная угроза |
| Сотрудник в плашке «отсутствует», но логировал часы | Период отсутствия охватывает сегодня, но он логировал задним числом или в первый день отпуска | Норма; отсутствие настраивается в Capacity, переписывать его не нужно |
| Виджет «Ворклоги по категориям» пуст, хотя ворклоги есть | У всех задач команды не проставлены категории — они в очереди «К разбору» | Раздел «Категоризация» → разобрать стек, проставить категории |
| Карточка сотрудника в части «Последний ворклог» красная, хотя он сегодня работал | Сегодняшние ворклоги ещё не синхронизировались | Запустить быструю синхронизацию в разделе «Синхронизация» |

## 8. Связанные разделы

**Откуда приходят данные:**
- **Синхронизация** (`/sync`) — без свежей синхронизации цифры заморожены.
- **Категоризация** (`/categories`) — задачи без категории не попадают в дашборд (часы есть, но висят в «Незаполненных worklog» или вообще не считаются).
- **Capacity** (`/capacity`) — план по нормированным работам считается отсюда (база ресурсов + правила распределения).
- **Планирование сценариев** (`/planning`, `/resource-planning`) — первый виджет показывает проекты только из утверждённого сценария квартала.
- **Глобальный фильтр команды** (в шапке) — режет всё.
- **Глобальный период** (в шапке) — выбирает квартал/месяц.

**Куда уходит дальше:**
- **Проекты** (`/projects`) — drill-in из первого виджета по клику на строку проекта.
- **Аналитика** (`/analytics`) — drill-in из всех трёх виджетов: по сотруднику, виду работ, категории.
- **Отчёт по видам работ** (`/analytics/work-type-report`) — drill-in из второго виджета по иконке диаграммы рядом с видом работ.

Главная — это не «отдельный отчёт», а сводка по уже существующим данным из других разделов. Если на дашборде цифры выглядят неправильно — починка всегда на одном из связанных разделов выше.
