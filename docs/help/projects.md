# Проекты

Раздел `/projects` — карточки квартальных проектов команды: левая колонка со списком, правая — детальная панель в одном из двух режимов. Здесь PM смотрит, что сделано по проекту, читает короткое ИИ-резюме и при необходимости готовит одностраничник для встречи или отчёта.

---

## 1. Зачем раздел

Дашборд и Аналитика показывают цифры по всей команде, по сотрудникам и по периодам. Но руководству и заказчику нужна не «средняя температура», а ответ по конкретному проекту: что мы делали, к какому результату пришли, кто вложился, как заказчик оценил. Эта страница собирает по каждому проекту единый паспорт — голы, ход работы, ворклог, оценки заказчика и короткое резюме от ИИ.

Раздел показывает только задачи с категорией «Квартальные задачи» или «Архив квартальных задач». Остальная работа (сопровождение, техдолг, встречи, инициативы) сюда не попадает — она живёт в Дашборде, Аналитике и других разделах.

---

## 2. Когда сюда заходить

- Перед встречей с руководством по проекту — посмотреть статус, резюме и топ-задачи.
- Перед встречей с заказчиком — открыть «Презентацию», распечатать или сохранить картинкой.
- Когда нужно понять, как загружены сотрудники именно по этому проекту (а не вообще по кварталу).
- Когда заказчик прислал оценку — проверить, что она подтянулась из Jira.
- Когда хочется быстро вспомнить, чем закончился проект прошлого квартала (вкладка «Архив квартальных задач»).
- Когда из Дашборда или Аналитики кликнули на ключ или название проекта — попадаем сюда.

## 3. Элементы интерфейса

Раздел построен по схеме «список слева — карточка справа». Список фильтруется по году, кварталу, статусу и категории; карточка переключается между двумя видами одной кнопкой.

### 3.1. Список проектов слева

| Элемент | Что делает |
|---------|-----------|
| Год | Меняет год отчётного периода. По умолчанию — текущий. |
| Q1 / Q2 / Q3 / Q4 | Бирюзовая плашка — выбранный квартал. По умолчанию — текущий. |
| Поиск | Ищет по названию проекта или ключу. С задержкой 0.3 сек, чтобы не дёргать сервер на каждом нажатии. |
| Статус | «Все» / «В работе» / «Готов». Срез по статусу проекта в Jira. |
| Категория | «Все» / «Квартальные» / «Архив». Срез по категории задачи в системе. |

Каждая карточка проекта в списке показывает: ключ Jira, название, период «месяц–месяц», количество часов команды, количество задач, количество участников, плашку статуса и среднюю оценку заказчика (звёздочки). Левая полоска карточки окрашена в цвет статуса (серый — новый, бирюзовый — в работе, зелёный — готов). Выбранная карточка подсвечена бирюзовой рамкой.

Клик по карточке открывает детальную панель справа и обновляет адрес страницы — ссылку можно скопировать и переслать.

### 3.2. Шапка детальной панели

Сверху панели — закреплённая шапка с названием проекта, ключом-ссылкой на Jira, периодом, статусом и подписью «AI-резюме обновлено …». Здесь же кнопки:

| Кнопка | Что делает |
|--------|-----------|
| Анализ / Презентация | Переключает вид правой панели (см. 3.3 и 3.4). |
| Обновить AI | Запускает перегенерацию ИИ-резюме именно по этому проекту прямо сейчас. Доступна, если в Настройках ИИ включён. |
| PNG | Сохраняет вид «Презентация» картинкой PNG. Если открыт «Анализ», то страница сама переключится на «Презентацию», подождёт прорисовку и сохранит файл `<КЛЮЧ>_ГГГГММДД.png` в загрузки. |
| ⋯ | Меню. Сейчас в нём «Копировать ссылку» — копирует прямую ссылку на проект в буфер. |

### 3.3. Вид «Анализ» (по умолчанию)

Аналитический вид — две колонки карточек.

Левая колонка:
- **Цели проекта** — короткие пункты из ИИ-резюме. Если ИИ ещё не сработал, показывается фрагмент описания из Jira.
- **На что ушло время** — диаграмма-«бублик» по категориям трудозатрат внутри проекта (или по группам работ от ИИ, если у проекта есть резюме). В центре «бублика» — общая сумма часов и сколько это недель команды. Слева от каждой группы — её доля.
- **Участники** — список сотрудников с долей и часами. Клик по сотруднику открывает Аналитику с фильтром «этот человек × этот проект».

Правая колонка:
- **Основной результат** — чек-лист от ИИ: что считается сделанным, что ещё нет.
- **Статус проекта** — короткий текстовый комментарий ИИ + KPI-плашки (задач всего, участников и т. п.).
- **Оценка заказчика** — звёздочки и числа 0–5 по трём измерениям: Качество, Скорость, Результат. Если в Jira ничего не заполнено — подпись «появится после заполнения полей в Jira».
- **Топ-3 задачи** — самые «тяжёлые» по ворклогу задачи. Клик по задаче — переход в Аналитику с фильтром по этой задаче.

### 3.4. Вид «Презентация»

Тот же проект, но одной длинной страницей в виде сторителлинга — для печати или для встречи. Идёт строго по разделам:

1. **Что мы делали** — пронумерованные цели + фрагмент описания из Jira.
2. **Какой результат** — чек-лист ✓/○ + комментарий ИИ.
3. **Кто работал** — горизонтальные полоски с часами и процентами по сотрудникам.
4. **На что ушло время** — крупный «бублик» с легендой и Топ-3 задач списком.
5. **Как оценили** — три блока со звёздочками: Качество, Скорость, Результат.

Этот вид специально сверстан так, чтобы помещаться в одну колонку шириной до 960 пикселей — подходит и для печати, и для PNG.

### 3.5. Кнопка справки

В шапке детальной панели справа — кнопка с вопросом, открывает эту справку. Видна и в режиме «Анализ», и в режиме «Презентация».

## 4. Откуда цифры

- **Список проектов** определяется категорией «Квартальные задачи» или «Архив квартальных задач», установленной в разделе «Категоризация задач». Без категории проект не появится в списке.
- **Год и квартал** — выбираются в шапке списка. Проект попадает в выбранный квартал по «периоду» (дата начала / дата окончания из Jira) или по дате последней активности.
- **Часы, участники, топ-задачи, разбивка по категориям** — считаются из ворклогов задач этого проекта, ограниченных глобальным фильтром команды (если он стоит) и квартальным окном. То есть видим именно ту часть проекта, которую вела наша команда.
- **Цели проекта, основной результат, статус, разбивка работ** — это ИИ-резюме. Перегенерация запускается по расписанию (в разделе «Синхронизация» → «Расписание») и вручную кнопкой «Обновить AI» в шапке проекта. Если в Настройках выключен ИИ, новых саммари не появится — будет показано последнее, что есть в кэше.
- **Какой провайдер ИИ работает** — выбирается админом в Настройках → ИИ. По расписанию резюме обновляются тем провайдером и моделью, которые там стоят на момент запуска.
- **Оценка заказчика** — три отдельных поля Jira (качество, скорость, результат), заполняются заказчиком в его задачке. Раздел показывает их «как есть», ничего не пересчитывая.
- **Описание проекта** — берётся из поля «Описание» в Jira (обрезается до ~600–800 символов для аккуратной верстки).
- **Топ-3 задачи** — три задачи проекта с самой большой суммой ворклога за выбранный квартал.

Все цифры обновляются только после синхронизации с Jira. Если в Jira что-то поменялось, а синхронизации не было — здесь это пока не увидим.

## 5. Типовые задачи

### Сценарий А: проверить статус по проекту перед встречей

1. Открыть `/projects`, выбрать год и квартал.
2. В поиске или в списке найти нужный проект, кликнуть на карточку.
3. В виде «Анализ» прочитать «Статус проекта» и «Основной результат» сверху справа.
4. Посмотреть «Цели проекта» и «На что ушло время» слева — понять, на что реально тратилось время.
5. Если ИИ-резюме старое (по подписи «AI-резюме обновлено …») — нажать «Обновить AI», подождать минуту и обновить страницу.

### Сценарий Б: распечатать одностраничник для заказчика

1. Открыть карточку проекта.
2. В шапке справа переключиться на «Презентацию» (или сразу нажать «PNG» — она это сделает за вас).
3. «PNG» — сохранится файл с именем `<КЛЮЧ>_ГГГГММДД.png`. Готово, можно вставлять в письмо или отчёт.
4. Если нужна распечатка — открыть «Презентацию» и распечатать страницу средствами браузера (Ctrl + P). Презентация специально свёрстана узкой колонкой, чтобы помещалась на лист.

### Сценарий В: понять, почему ИИ дал такой вывод

1. В виде «Анализ» сравнить ИИ-цели с описанием проекта (в той же карточке).
2. Сравнить чек-лист «Основной результат» с реальными статусами задач (вкладка «Топ-3» + Аналитика).
3. Если резюме явно расходится с фактами — нажать «Обновить AI» в шапке проекта. Свежий запуск возьмёт текущее состояние задач и описаний.
4. Если и после этого ИИ говорит ерунду — проверить в Настройках, какой провайдер и модель работают, и нет ли в описании проекта/задач устаревшего текста, который сбивает ИИ.

### Сценарий Г: отметить новые риски

1. Открыть проект → «Анализ» → «Статус проекта» — здесь ИИ может прямо сказать, что проект «под риском».
2. Если в ИИ-резюме риска нет, а вы его видите — проверить топ-задачи (вкладка справа): если в Топ-3 болтаются открытые задачи без движения, это и есть сигнал.
3. Договориться с заказчиком о решении. После того, как ситуация изменится в Jira, нажать «Обновить AI» — резюме перегенерируется уже с новыми вводными.

## 6. Ограничения

- В списке видны **только** проекты с категорией «Квартальные задачи» или «Архив квартальных задач». Если проекта нет в списке — почти всегда дело в неправильной (или отсутствующей) категории.
- Фильтры по году/кварталу/статусу/категории и поиск **не сохраняются** между сессиями.
- Выбранный режим (Анализ / Презентация) **сохраняется в адресе страницы** — ссылка на «Презентацию» приведёт нового пользователя сразу в режим презентации.
- ИИ-резюме показывает то, что было сгенерировано последний раз. Если ИИ выключен или провайдер не отвечает, новые саммари не появятся — будет видно старое.
- «Обновить AI» работает только когда ИИ глобально включён в Настройках. Если выключен — кнопка не сработает.
- «PNG» рендерит только вид «Презентация». В режиме «Анализ» кнопка сначала автоматически переключит на «Презентацию», подождёт прорисовку и потом сохранит файл.
- «Оценка заказчика» — это **показ как есть** из трёх полей Jira. Если заказчик не заполнил — будет прочерк. Раздел сам ничего не считает и не усредняет (кроме строки в списке слева — там среднее по заполненным).
- Глобальный фильтр команды (в шапке приложения) влияет на часы, участников и Топ-3 задач — это всегда срез «нашей команды по этому проекту», а не весь проект целиком.

## 7. Частые проблемы

| Симптом | Причина | Что делать |
|---------|---------|------------|
| Нет ИИ-резюме (Цели / Результат / Статус пустые) | ИИ выключен в Настройках, либо провайдер не отвечает | Настройки → ИИ → включить, проверить ключ и провайдера; нажать «Обновить AI» в шапке проекта |
| Резюме явно устарело (упоминает задачи, которые уже закрыты) | Резюме обновляется по расписанию + после полной синхронизации. Между запусками оно «застывшее» | Нажать «Обновить AI» в шапке проекта; либо дождаться очередного запуска расписания (см. раздел «Синхронизация → Расписание») |
| Проекта нет в списке | Категория задачи в системе — не «Квартальные» и не «Архив квартальных» | Открыть «Категоризация задач», найти задачу, поставить категорию «Квартальные задачи» (или «Архив квартальных задач») и сохранить |
| Списка проектов вообще нет | Не выбран квартал / неправильный год; либо ни одна задача не подходит под Scope и фильтр команды | Проверить год и квартал сверху списка; в Настройках → Scope убедиться, что проекты Jira добавлены |
| «Оценка заказчика» — прочерк | Заказчик ещё не заполнил поля «Качество / Скорость / Результат» в Jira | Подождать заполнения; либо проверить в Настройках → Поля Jira, что три «оценочных» поля правильно указаны |
| Часы и участники меньше, чем ожидалось | Стоит глобальный фильтр команды — он режет ворклоги до сотрудников этой команды | Снять или сменить команду в шапке приложения; сравнить с разделом «Аналитика» по тому же проекту |
| «PNG» сохранил пустую или обрезанную картинку | Не успели прорисоваться диаграммы (анимация Recharts) | Открыть «Презентацию» вручную, подождать пару секунд, ещё раз нажать «PNG» |
| Кнопка «Обновить AI» не реагирует (серая) | ИИ выключен глобально в Настройках | Настройки → ИИ → включить |
| После «Обновить AI» резюме не поменялось | Запрос ушёл в очередь и ещё не завершился; либо провайдер вернул то же самое (нет новых данных) | Подождать минуту, обновить страницу. Если повторяется — посмотреть в Настройках → ИИ, какой провайдер используется и есть ли ошибки |

## 8. Связанные разделы

**Откуда приходят данные:**
- **Синхронизация** (`/sync`) — без свежей синхронизации задачи, ворклоги и оценки заказчика не обновятся.
- **Синхронизация → Расписание** — там же стоит cron-задача автоматической перегенерации ИИ-резюме проектов.
- **Категоризация задач** (`/categories`) — определяет, что вообще попадает в этот раздел: только «Квартальные задачи» и «Архив квартальных задач».
- **Настройки → Scope** — какие проекты Jira синхронизируются.
- **Настройки → Поля Jira** — какие поля Jira считаются «оценками заказчика», «командой» и «целями».
- **Настройки → ИИ** — провайдер и модель ИИ, общий «выключатель» резюме.
- **Глобальный фильтр команды** (в шапке приложения) — режет часы, участников и Топ-3 по этому проекту.

**Куда уходит результат:**
- **Дашборд** (`/`) — KPI и виджеты пересекаются с цифрами этого раздела по тем же квартальным задачам.
- **Аналитика** (`/analytics`) — клик по сотруднику или по задаче из карточки проекта открывает Аналитику с уже выставленным фильтром.
- **Executive Dashboard** (`/executive`) — KPI верхнего уровня по квартальным проектам.
- **Презентация / печать** — PNG-картинка или печать страницы — для встреч и отчётов руководству / заказчику.
