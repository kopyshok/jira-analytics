# Аналитика трудозатрат

Раздел `/analytics` — иерархический отчёт по часам. Показывает, кто, сколько и над чем работал за период. Это главный аналитический разрез по факту работы: команды → роли → сотрудники → виды работ → категории → задачи. Все цифры — фактические списания часов из Jira, очищенные через категоризацию и фильтр команды.

---

## 1. Зачем раздел

В Jira ежедневно копится сотни ворклогов от десятков сотрудников. Сводных отчётов по часам в Jira нет — она показывает либо одну задачу, либо одного сотрудника. Этот раздел собирает все списания за период в один отчёт и даёт пройтись по иерархии сверху вниз: команда целиком → конкретная роль → конкретный сотрудник → конкретный вид работ → конкретная категория → конкретная задача → конкретные ворклоги.

Главные вопросы, на которые отвечает раздел:

- Сколько часов команда отработала за квартал и сколько было запланировано?
- Кто перерабатывает, кто недогружен?
- Куда уходит время — в продуктовую работу, во встречи, в техдолг, в коммуникации?
- В каких задачах конкретного сотрудника зашит «провал» по плану?
- Списываются ли сотрудники моей команды на чужие задачи (за периметр команды)?

Любая цифра в дашборде и в виджете — это агрегат, который раскладывается именно здесь.

---

## 2. Когда сюда заходить

- Утренний разбор перегруза — кто из сотрудников зашкаливает за норму, в каких категориях.
- Подготовка отчёта руководству за квартал — сколько часов в каждой категории, кто во что играл.
- Когда в дашборде или в карточке проекта подозрительная цифра — нажать на неё и провалиться сюда.
- Расследование «куда у Иванова ушло 180 часов» — открыть его строку, провалиться до задач, открыть ворклоги.
- Проверка чужих списаний — мои сотрудники списались на задачи других команд, надо понять масштаб.
- План/факт по виду работ — сколько из 200 часов на «Разработку» реально сожрала разработка.

## 3. Элементы интерфейса

Раздел построен по принципу мастер-деталь: слева — список команд, посередине — иерархическая таблица, в шапке — глобальные настройки периода и фильтры. Все цифры в таблице реактивно перестраиваются при смене любого фильтра.

### 3.1. Шапка и глобальный период

Период выбирается в шапке приложения (не в самом разделе): год, квартал, при необходимости — конкретный месяц. Этот период действует одновременно на дашборд, аналитику, отчёт по видам работ, capacity и executive — переключение года в одном месте перестраивает все разделы.

Рядом с глобальным периодом в шапке самого раздела стоит «уточнение по датам» — обычный диапазон дат «с — по». Используйте его, когда нужно посмотреть не весь квартал, а конкретный отрезок (например, прошлую неделю или конкретный спринт). Уточнение перекрывает глобальный период.

### 3.2. Переключатель «Текущий вид / Новый вид»

Два варианта раскладки экрана:

- **Текущий вид** — классический вид: KPI-плитки сверху, список команд слева, иерархическая таблица справа. По умолчанию.
- **Новый вид** — мастер-деталь: список команд слева, таблица посередине, справа — панель деталей выбранной задачи (контекст, описание, цели, ворклоги, кнопка «В Jira»).

Запомнится в браузере между визитами.

### 3.3. KPI-плитки

Над таблицей пять плиток с итогами за выбранный период (с учётом фильтра команды и всех уточнений):

| Плитка | Что показывает |
|--------|---------------|
| Σ Часов факт | Сумма всех списаний |
| Σ Часов план | Норма по сотрудникам за период, очищенная от отсутствий |
| % Выполнения | Факт / план. Жёлтый при <70%, зелёный при норме, красный при >110% |
| Сотрудников | Число сотрудников, которые что-то списали |
| Чужих часов | Сколько часов и % списано на задачи команды, которая не своя для сотрудника. Оранжевый, если есть |

KPI-плитки — это «верхняя строка таблицы», но крупно и без подгрупп. Они меняются вместе с фильтрами.

### 3.4. Список команд (мастер)

Колонка слева. Показывает все команды, которые попали под глобальный фильтр команды (тот, что в шапке приложения). Рядом с именем команды — её итоговые часы факт. Клик по команде — оставить в таблице только её. «Все команды» — снять локальный выбор.

Если команд в глобальном фильтре несколько и выбрана конкретная — в таблице автоматически пропадает уровень «Команда» (всё равно одно значение). При «Все команды» команды остаются верхним уровнем иерархии.

### 3.5. Иерархическая таблица

Шесть уровней группировки сверху вниз: **Команда → Роль → Сотрудник → Вид работ → Категория → Задача**. Каждый уровень — это разворачивающаяся строка с агрегатами вниз по дереву. На уровне задачи в новом виде можно ещё провалиться до конкретных ворклогов.

Колонки таблицы:

| Колонка | Что показывает |
|---------|---------------|
| Группа / Задача | Имя группы или ключ + заголовок задачи; для задачи — статус-тег, цветная плашка «Чужая», ссылка в Jira |
| Часы факт | Сумма списаний под этим узлом; подсветка по цвету выполнения |
| Часы план | Норма для этого узла (только когда есть смысл — сотрудник × вид работ) |
| % план | Факт / план, цвет по тому же правилу |
| % в группе | Доля строки в её непосредственной родительской группе |
| % от итога | Доля строки от общего факта команды/среза |
| Ворклогов | Сколько списаний попало под узел |
| Задач | Сколько уникальных задач |
| Сотр. | Сколько сотрудников засветилось под этим узлом |
| Ср. мин | Средняя длина одного списания, в минутах |

Минимально показываются «Группа / Задача» и «Часы факт», всё остальное настраивается (см. 3.7).

### 3.6. Переключение группировки (DnD и пресеты)

Уровни группировки переставляются и скрываются под пользователя. Открыть «Настройка отчёта» в шапке → секция «Группировка»:

- **Четыре пресета** одним кликом: «Стандарт» (все шесть уровней), «По людям» (сотрудник → категория → задача), «По категориям» (категория → вид работ → задача), «По видам работ» (вид работ → категория → задача).
- **Drag-and-drop** — двигаем уровни вверх-вниз, меняем порядок (например, сначала Категория, потом Сотрудник).
- **Глазок** — скрываем уровень, не удаляя его. Скрытый уровень схлопывается, агрегаты собираются на уровень выше.

Уровень «Задача» всегда виден и всегда последний — это лист дерева. Все настройки запоминаются за пользователем на сервере, не сбрасываются между сессиями.

### 3.7. Управление видимостью колонок

Та же модалка «Настройка отчёта» → секция «Столбцы». Чекбоксы по каждой необязательной колонке (Часы план, % план, % в группе, % от итога, Ворклогов, Задач, Сотрудников, Ср. минут). Настройка сохраняется за конкретным пользователем — у разных аналитиков может быть свой набор. Колонки «Группа / Задача» и «Часы факт» всегда видны.

Чуть ниже — переключатель **«Полоска заполнения в колонке Часы факт»**: тонкая полоса под цифрой показывает долю строки в текущей группе (или от общего итога на корне). Помогает на глаз сравнивать сёстры в одной группе.

### 3.8. Метки на строках

- **Статус-тег задачи** — цвет берётся из Jira (новая / в работе / готова / отменена).
- **Плашка «Чужая»** (оранжевая) — задача принадлежит чужой команде, сотрудник списался на неё «за периметр». Видно сразу под ключом задачи и блоком — указание команды-владельца.
- **Плашка с командой-владельцем** — под строкой задачи. Указывает имя команды, к которой относится задача, и цветной квадрат (цвет фиксированный по имени команды).
- **Оранжевый чип «⚠ N · Mч · X%»** на групповых строках — суммарный «след» чужих задач под этим узлом: сколько штук, сколько часов, какой процент от факта группы. Появляется только если в группе есть чужие задачи.

### 3.9. Бары и доли

Колонка «Часы факт» опционально показывает тонкую полоску под цифрой — это визуальная доля строки в её группе. Помогает на длинных списках видеть «жирные» строки без сортировки. Включается одним чекбоксом в настройках отчёта.

Колонка «% в группе» — точное значение для текущего уровня (например, у сотрудника «% в группе» = его доля в команде).

### 3.10. Drill-in с виджетов

В аналитику можно «провалиться» из других разделов — открывается уже с проставленным фильтром:

- **Дашборд → виджет «Норма» (NormWorkWidget)** — клик по сотруднику или категории.
- **Дашборд → виджет «Категории» (CategoryWidget)** — клик по сотруднику или категории.
- **Карточка проекта → блок «Категории»** — клик по дольке.
- **Карточка проекта → блок «Сотрудники»** — клик по строке сотрудника.
- **Карточка проекта → блок «Топ задач»** — клик по задаче.

Во всех случаях аналитика открывается с заранее проставленным фильтром (сотрудник, категория, проект, задача) — настройки группировки и колонок остаются такие же, как у вас были.

### 3.11. Фильтры над таблицей

Над таблицей — четыре фильтра:

- **Сотрудник** — оставить в отчёте только одного сотрудника. Поиск по имени.
- **Поиск задачи** — по ключу или по подстроке в заголовке.
- **Вид работ** — оставить только списания по конкретному виду (например, «Разработка» или «Анализ»).
- **Категория** — оставить только задачи указанной категории (например, «Техдолг» или «Сопровождение»).

Фильтры запоминаются в адресной строке — можно скопировать ссылку и переслать коллеге с тем же срезом.

### 3.12. Панель «Детализация задачи» (только Новый вид)

В новом виде справа открывается панель с детальной информацией по выбранной задаче:

- Ключ, заголовок, статус, тип задачи в Jira.
- Описание, цели задачи (фиолетовые теги, если поле «Цели» заполнено).
- Контекст: родительские задачи, дочерние задачи (можно провалиться по дереву).
- Блок «Категоризовать» — поставить или поменять категорию, не уходя из аналитики.
- Список ворклогов за выбранный период.
- Кнопка «В Jira» — открыть задачу на atlassian.net в новой вкладке.

### 3.13. Экспорт XLSX

Кнопка «Экспорт XLSX» в шапке — выгружает текущий срез отчёта в Excel со всеми колонками, которые вы оставили видимыми, и с текущей группировкой. Удобно отдавать руководству как «снимок» отчёта на конкретную дату.

## 4. Откуда цифры

- **Источник часов** — ворклоги Jira, синхронизированные через раздел «Синхронизация» (см. подраздел «Дополнительно → Обновить ворклоги»). Сырые списания хранятся в локальной БД.
- **Привязка к сотруднику** — список сотрудников из Jira; сотрудник попадает в команду через настройки команды (раздел Capacity → Команды).
- **Привязка к проекту/задаче** — стандартная иерархия задач Jira (Parent Link) с учётом правил иерархии из настроек.
- **Категория** — берётся из раздела «Категоризация задач» (`/categories`). Если у задачи нет своей категории, наследуется от ближайшего предка. Задачи без категории и без наследования в аналитику не попадают.
- **Вид работ** — берётся из поля «Вид работ» (custom field в Jira) или, если не проставлен, выводится по правилам категории (по умолчанию все списания на задачу относятся к виду работ, который привязан к её категории).
- **Роль сотрудника** — берётся из карточки сотрудника (раздел Capacity → Сотрудники). Один сотрудник = одна роль. Если не указана — попадает в группу «Без роли».
- **Команда задачи** — берётся из поля Jira «Команда» (основная или участвующая, настройка — `Settings → Поля Jira`). Если команда задачи не совпадает с командой сотрудника, который списался, такая задача помечается как «Чужая».
- **План** — норма часов сотрудника за период за вычетом отсутствий и нерабочих дней (см. раздел Capacity и Производственный календарь). Считается по комбинации сотрудник × вид работ. На более высоких уровнях план — сумма планов по всем подгруппам.
- **Глобальный фильтр команды** (в шапке приложения) — обрезает выборку до выбранных команд. Сотрудник не из выбранных команд → его списания в отчёт не попадают.
- **% в группе** — доля строки в её непосредственной родительской группе. Например, у сотрудника «% в группе» = его доля в роли (если выше «Роль»), у задачи — доля в категории.
- **% от итога** — доля строки от общего факта всего отчёта (с учётом фильтров).

## 5. Типовые задачи

### Сценарий А: утренний разбор перегруза

1. Открыть `/analytics`.
2. В шапке приложения убедиться, что стоит ваша команда (или несколько).
3. Глобальный период — текущий квартал.
4. KPI-плитка «% Выполнения» — посмотреть общий статус.
5. Открыть настройки отчёта → пресет «По людям». Группировка станет «Сотрудник → Категория → Задача».
6. Отсортировать таблицу по «% план» — сверху окажутся перегруженные.
7. Развернуть строку перегруженного сотрудника, посмотреть, в какой категории идёт перерасход.
8. Развернуть категорию — увидеть конкретные задачи. Если все часы в «Сопровождении» — обсудить с командой нагрузку по сопровождению.

### Сценарий Б: отчёт по конкретному проекту

1. Открыть карточку проекта (`/projects/{ключ}`).
2. В блоке «Категории» или «Сотрудники» провалиться на нужный срез — откроется аналитика уже с фильтром по проекту.
3. В аналитике поставить пресет «По категориям» или «По людям» в зависимости от того, что нужно показать.
4. Уточнить период через «с — по», если квартал слишком широкий.
5. «Экспорт XLSX» — отдать заказчику.

### Сценарий В: настроить свой набор колонок

1. Кнопка «Настройка отчёта» в шапке раздела.
2. Секция «Столбцы» — снять то, что не нужно (например, «Ср. минут», «Сотрудников»).
3. Оставить только «Часы факт», «% план», «% от итога» — если нужен компактный вид.
4. Включить «Полоска заполнения в колонке Часы факт» — будут видны доли визуально.
5. Закрыть модалку — настройка сохранится за вашим пользователем и подхватится при следующем визите.

### Сценарий Г: drill-in с дашборда

1. На главной странице (`/`) в виджете «Категории» видна аномалия — большая доля «Техдолга».
2. Клик по дольке «Техдолг» — открывается аналитика с фильтром «category=tech_debt».
3. В таблице — список сотрудников, которые писали в техдолг.
4. Развернуть конкретного сотрудника → посмотреть, какие именно задачи он закрывал.
5. Кликнуть строку задачи → справа откроется панель «Детализация» (если выбран Новый вид) с описанием и ворклогами.
6. Если нужно перейти к самой задаче — кнопка «В Jira» в панели.

### Сценарий Д: посмотреть «чужие» списания

1. KPI-плитка «Чужих часов» оранжевая — есть списания за периметр команды.
2. Включить группировку «По людям».
3. Развернуть сотрудника — на строках задач увидеть оранжевые плашки «Чужая» и под ними имя чужой команды.
4. Оранжевый чип на группе показывает суммарный объём чужих часов под узлом.
5. Если чужие задачи имеют смысл — добавить сотрудника во вторую команду через настройки. Если нет — обсудить с сотрудником, на что он списался.

## 6. Ограничения

- **Задачи без категории не попадают в отчёт.** Если ворклог есть, а задача висит в «К разбору» в `/categories` — этого ворклога в аналитике не будет. Сначала разобрать категории.
- **План считается только по комбинации сотрудник × вид работ.** На уровне команды или роли «Часы план» — это сумма планов подгрупп, а не отдельная норма команды.
- **Глобальный фильтр команды действует до выборки.** Если убрать команду из фильтра — её часы исчезнут везде, включая «Чужие». Чужими считаются задачи относительно «своих» сотрудников, а не наоборот.
- **Уточнение по датам** «с — по» перекрывает глобальный период, но не отменяет фильтр команды.
- **Пресеты группировки** перезаписывают и порядок, и видимость уровней. Если есть кастомная настройка — сохраните её визуально, потому что кнопка пресета сбросит её.
- **Колонка «Часы план»** показывает «—» для строк, где нет смысла плана (например, для задачи отдельно — план есть только на уровне сотрудник × вид работ).
- **«Чужие задачи»** определяются по полю «Команда» из Jira. Если поле не настроено или не заполнено — плашка не появится.
- **Drill-in с виджетов** добавляет фильтр, но не сбрасывает остальные. Если у вас уже стоял фильтр по сотруднику — он останется, дополнительный фильтр приплюсуется.

## 7. Частые проблемы

| Симптом | Причина | Что делать |
|---------|---------|------------|
| Цифры в аналитике не совпадают с дашбордом | Период разный (в шапке выбран другой квартал) или включён фильтр в адресной строке | Сверить глобальный период в шапке. Снять фильтры — кнопка «Сбросить фильтры» (появляется при пустой выборке) |
| Сотрудник в отчёте есть, а часы пустые | Все его задачи без категории — попали в «К разбору» в /categories | Открыть `/categories`, разобрать его задачи, вернуться и обновить страницу |
| Не вижу нужной колонки | Колонка скрыта в персональных настройках | «Настройка отчёта» → «Столбцы» → включить |
| KPI «Чужих часов» большое, но не вижу плашек | Сейчас включён пресет, где уровень «Команда» скрыт, а задачи свёрнуты под группами | Развернуть до задач или переключиться на «Стандарт» |
| Таблица пустая | Глобальный фильтр команды отрезает всё; либо период вне рамок данных | Расширить фильтр в шапке или сменить период |
| План = «—» на верхних уровнях | Нет норм у сотрудников под этой группой | Capacity → проверить, заполнены ли нормы и отсутствия |
| Открыл аналитику с виджета, но фильтр странный | Виджет передал параметр (например, `?category=...`), который добавился к уже стоящему | Сбросить фильтры или почистить адресную строку |
| «% в группе» = 100% у всех строк | Группа состоит из одной строки или включён пресет, где промежуточные уровни скрыты | Это нормально — % считается относительно непосредственного родителя |
| После перетаскивания уровней пресет «потерялся» | DnD автоматически переводит конфигурацию в режим «custom» | Нажать кнопку пресета снова — он перезапишет порядок |
| Экспорт XLSX отдаёт «не то» | Экспорт использует текущие фильтры и набор колонок | Перед экспортом проверить — стоят ли нужные фильтры и видны ли нужные колонки |

## 8. Связанные разделы

**Откуда приходят данные:**

- **Синхронизация** (`/sync`) — без свежей синхронизации в аналитике не будет новых ворклогов и задач из Jira. Обновление ворклогов — вкладка «Дополнительно».
- **Категоризация задач** (`/categories`) — задача без категории не попадает в аналитику.
- **Capacity** (`/capacity`) — нормы сотрудников, отсутствия, производственный календарь. Отсюда берётся «План».
- **Настройки → Поля Jira** — какое поле в Jira считается «Командой» и «Видом работ».
- **Настройки → Категории работ** — справочник категорий и связка категория ↔ вид работ.
- **Глобальный фильтр команды** (шапка приложения) — определяет, какие сотрудники попадут в отчёт.
- **Глобальный период** (шапка приложения) — год + квартал + месяц.

**Куда уходит результат:**

- **Дашборд** (`/`) — виджеты «Норма» и «Категории» дают drill-in в аналитику.
- **Отчёт по видам работ** (`/analytics/work-type-report`) — тематический отчёт по конкретному виду работ; можно провалиться из строки «Вид работ» по стрелочке.
- **Проекты** (`/projects/{ключ}`) — карточка проекта тоже даёт drill-in.
- **Executive Dashboard** (`/executive`) — топовые KPI приходят отсюда.
- **XLSX-экспорт** — снимок отчёта для отправки руководству.

Это самый «нагруженный» отчёт в системе. Любая аналитическая претензия («у нас всё плохо с техдолгом», «Иванов недогружен», «команда выпиливает чужие задачи») проверяется и подтверждается именно здесь — провалом по дереву до конкретной задачи и конкретного ворклога.
